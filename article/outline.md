# AIエージェントに3時間1万ドルを預けてみた

## 記事構成

### 1. 導入: 実験の概要
- なぜこの実験をやるのか？ AIの自律性の限界を探る
- 前提条件: $10,000のAzureクレジット、3時間の制限時間
- ゴール: AIエージェントチームが自律的にプロダクトを構築できるか
- 読者への問いかけ: 「あなたならAIに1万ドルを預けますか？」

### 2. セットアップ: MAGIシステムとは
- エヴァンゲリオンのMAGIシステムからのインスピレーション
  - メルキオール（科学者）、バルタザール（母）、カスパー（女）
  - 3つの人格が合議して意思決定を行う仕組み
- 技術スタック
  - Azure OpenAI (GPT-5, GPT-5.1, GPT-5.2, gpt-image-1.5, sora-2)
  - Azure Static Web Apps, Azure Storage
  - Claude Code (エージェント実行環境)
- アーキテクチャ図

### 3. チーム編成: 3つのAIエージェントの役割
- **MELCHIOR (メルキオール/科学者)**
  - バックエンドAPI・合議エンジン構築担当
  - Python FastAPIでMAGIコンセンサスシステムを実装
  - GPT-5の推論能力を活用した多角的意思決定
- **BALTHASAR (バルタザール/母)**
  - フロントエンド・UXデザイン担当
  - インタラクティブWebデモの構築
  - リアルタイムの合議プロセス可視化
- **CASPER (カスパー/女)**
  - 実験・監視・ドキュメント担当
  - コスト監視とリソース管理
  - 記事素材（画像、実験結果）の準備

### 4. 実行ログ: 30分ごとの報告
- **06:50 - キックオフ**: 各エージェントの起動と初期タスク割り振り
- **07:20 - 第1回スタンドアップ**: 基盤構築の進捗
- **07:50 - 第2回スタンドアップ**: API実装とUI設計
- **08:20 - 第3回スタンドアップ**: 統合テスト開始
- **08:50 - 第4回スタンドアップ**: デバッグと最適化
- **09:20 - 第5回スタンドアップ**: 最終仕上げ
- **09:50 - 最終報告**: 成果物の確認と記事執筆

### 5. 成果物: 何ができたか
- MAGIコンセンサスエンジン（バックエンドAPI）
- インタラクティブWebデモ
- 生成された画像・映像コンテンツ
- デプロイされたAzureリソース一覧
- スクリーンショット・デモ動画

### 6. コスト分析: $10,000の使い道
- APIコール別コスト内訳
  - GPT-5: トークン使用量と費用
  - GPT-5.1/5.2: 比較実験の費用
  - gpt-image-1.5: 画像生成費用
  - sora-2: 動画生成費用（もし使った場合）
- Azureインフラコスト
  - Storage Account
  - Static Web App
  - OpenAI リソース
- 時間帯別の消費推移グラフ
- ROI分析: 人間がやったら何日かかるか

### 7. 考察: AIに自律性を与えるとどうなるか
- うまくいったこと
  - 並列作業の効率性
  - コード品質（テスト・ドキュメント）
  - 一貫性のあるアーキテクチャ設計
- 課題と限界
  - エージェント間コミュニケーションのオーバーヘッド
  - 予期しないエラーハンドリング
  - 創造性 vs 安全性のトレードオフ
- AIエージェントの未来
  - マルチエージェント協調の可能性
  - 人間の監督の必要性
  - コスト効率の展望

### 8. まとめ
- 3時間で達成できたこと・できなかったこと
- $10,000は有効に使えたか
